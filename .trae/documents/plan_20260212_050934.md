The error `-bash: !': event not found` happens because `!` is a special character in bash (history expansion). When you tried to run the command with the password containing `!`, bash tried to interpret it.

### Plan: Reset Password Correctly

To avoid the bash error, we need to wrap the password in single quotes strictly (which you did, but bash can still be tricky with `!` in interactive mode) or escape it.

However, the cleanest way is to let `mysql` prompt for the password or run the command inside the container interactively.

**Option 1: Interactive Mode (Safest)**
Run this command on the server:

```bash
docker exec -it jobportal-mysql-prod mysql -u root -p
```

* When prompted for password, enter: `Zpluse@2024!`

* Then, inside the MySQL shell `mysql>`, paste:

  ```sql
  ALTER USER 'jobportal'@'%' IDENTIFIED BY 'Zpluse@2024!';
  FLUSH PRIVILEGES;
  EXIT;
  ```

**Option 2: One-Liner (Escaped)**
If you prefer a one-liner, you must escape the `!` or disable history expansion:

```bash
set +H
docker exec -it jobportal-mysql-prod mysql -u root -p'Zpluse@2024!' -e "ALTER USER 'jobportal'@'%' IDENTIFIED BY 'Zpluse@2024!'; FLUSH PRIVILEGES;"
set -H
```

**After Resetting Password:**
Restart the backend to pick up the connection:

```bash
cd /opt/jobportal
docker-compose -f docker-compose.prod.yml restart backend
```

**Verify:**

```bash
curl http://localhost:8080/actuator/health
```

Please try **Option 1** as it is less prone to shell escaping errors.
