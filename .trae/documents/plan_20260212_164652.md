The deployment failed because the user configured in your GitHub Secrets (`SERVER_USER`) **does not have permission** to write files to the `/opt/jobportal/` directory on your server.

This is why you see: `scp: dest open "/opt/jobportal/docker-compose.prod.yml": Permission denied`.

### **The Fix**
Since you have **root access** in your server terminal (as seen in your screenshot), you can fix this easily by changing the ownership of that directory.

I need you to run a command on your server terminal. But first, check your **GitHub Secrets** to confirm what username you set for `SERVER_USER` (it's likely something like `ubuntu`, `jobportal`, or a specific user you created).

### **Commands to Run on Server**
Replace `YOUR_DEPLOY_USER` with the actual username from your `SERVER_USER` secret.

1.  **Change Ownership:**
    ```bash
    chown -R YOUR_DEPLOY_USER:YOUR_DEPLOY_USER /opt/jobportal
    ```
    *(Example: if your `SERVER_USER` is `ubuntu`, run `chown -R ubuntu:ubuntu /opt/jobportal`)*

2.  **Ensure Write Permissions:**
    ```bash
    chmod -R 755 /opt/jobportal
    ```

**After running these commands:**
1.  Go back to GitHub Actions.
2.  Click **"Re-run jobs"** on the failed deployment.
3.  It should succeed this time!
