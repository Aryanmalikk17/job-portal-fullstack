### **The Root Cause**
The file `DB_Scripts/00-create-user.sql` is hardcoding the user creation with the **WRONG host** and **WRONG password**:

```sql
CREATE USER 'jobportal'@'localhost' IDENTIFIED BY 'jobportal';
GRANT ALL PRIVILEGES ON * . * TO 'jobportal'@'localhost';
```

1.  **Wrong Host:** It creates the user for `'localhost'`, but your backend connects from a different container (e.g., `172.18.0.4`), so MySQL rejects the connection. It needs to allow access from `'%'` (any host).
2.  **Wrong Password:** It sets the password to `'jobportal'`, but your `.env` file (from GitHub Secrets) is likely setting it to something secure (like `Zpluse@2024!`).

Since `docker-compose` runs this script during initialization, it **overwrites** or conflicts with the user creation logic of the official MySQL image, leaving you with a broken user that can't connect from the backend.

### **The Solution**
We need to **modify this script** to allow connections from any host (`%`) and use the correct password, or better yet, rely on Docker's built-in environment variables which we already fixed.
