Since the deployment failed with "Permission denied" when trying to write a file, we know that your GitHub Action is connecting as a **non-root user** (likely `ubuntu`), but the folder `/opt/jobportal` is owned by `root`.

To find the correct username, we just need to list the users on your server.

### **Steps to Fix Permissions**

Run these commands in your server terminal (where you are logged in as `root`):

1.  **Find the username:**
    ```bash
    ls /home
    ```
    *   **If you see `ubuntu`**: Then your username is **`ubuntu`**.
    *   **If you see something else**: That is your username.

2.  **Fix the permissions (assuming the user is `ubuntu`):**
    *(If your user from step 1 is different, replace `ubuntu` with that name)*
    ```bash
    chown -R ubuntu:ubuntu /opt/jobportal
    chmod -R 755 /opt/jobportal
    ```

3.  **Retry Deployment:**
    *   Go back to GitHub Actions.
    *   Click **Re-run jobs** on the failed deployment.

This will allow the deployment user to write the necessary files to that folder.
