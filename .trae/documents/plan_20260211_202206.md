The error `fatal: detected dubious ownership in repository at '/opt/jobportal'` occurs because the owner of the directory `/opt/jobportal` is different from the current user (`root`). Git prevents operations in such directories for security reasons.

### Plan: Fix Git Ownership and Deploy

Since you are logged in as `root`, you need to explicitly tell Git to trust this directory.

**1. Fix Git Ownership (Run on Server)**
Execute this command to whitelist the directory:
```bash
git config --global --add safe.directory /opt/jobportal
```

**2. Retry Git Commands (Run on Server)**
Now that Git trusts the directory, run the pull commands again:
```bash
cd /opt/jobportal
git remote add origin https://github.com/Aryanmalikk17/job-portal-fullstack.git || echo "Remote exists"
git fetch origin
git reset --hard origin/main
```

**3. Deploy (Run on Server)**
```bash
docker-compose -f docker-compose.prod.yml up -d --build app
```

**4. Verify (Run on Server)**
```bash
curl http://localhost:8080/actuator/health
```

Please run these commands in your server terminal.
